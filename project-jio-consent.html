01 — Project Overview

India's first unified consent experience

India's Digital Personal Data Protection (DPDP) Act, enacted in 2023, requires companies like Jio to provide users with clear, transparent, and granular control over how their personal data is collected and used. Before this project, Jio had no unified consent experience — each product (JioMart, JioCinema, JioCare, etc.) handled consent independently, leading to inconsistent UX, legal risk, and fragmented data governance.

As the lead UX designer, I owned the end-to-end design of a centralised Consent Management Platform covering four distinct products: a consumer Marketing Page, a User Portal, an Admin Portal for enterprise operators, and TruConnect for B2B partners.

02 — The Problem

Regulatory context — DPDP Act 2023

India's Digital Personal Data Protection Act (2023) fundamentally changed how companies must handle user data. Under the Act, every data fiduciary (like Jio) is required to obtain explicit, informed, and purpose-specific consent from users before collecting or processing their personal data.

The Business Problem
  • Legal risk — inconsistent consent handling could violate DPDP requirements
  • Fragmented data governance — no single view of what users had consented to
  • Poor UX — users encountered different consent flows across Jio products
  • Duplicated effort — every product team building consent from scratch
The User Problem
  • Had no single place to see what data they had consented to share
  • Could not easily withdraw or update their consent preferences
  • Were unaware of parental controls for minor users
  • Received cookie banners inconsistently across Jio's web products
  • Had no grievance mechanism if their consent rights were violated

05 — Module 2
Module 02
User Portal
✓ Complete Dev File Ready

The User Portal is a self-service interface where Jio users can manage all aspects of their consent — from accepting and withdrawing permissions, to raising grievances and managing parental controls. It is the most comprehensive module, covering 8 complete user flows, all delivered as Figma dev files.

8 Complete User Flows

Flow Description
01Accept Consent User views and accepts data collection permissions — primary onboarding flow
02Withdraw Consent User withdraws previously granted permissions with clear confirmation steps
03Update Consent User updates the scope or purpose of their consent preferences
04Approve Pending Request User reviews and approves a pending consent request from a Jio service
05Raise Grievance Request User files a formal grievance if they believe their data rights were violated
06Cookie Banner & Preferences User manages cookie consent through a banner and a detailed preference center
07Parental Journey Parent/guardian manages consent on behalf of a minor user — the most complex flow
08Auto Renewal User manages automatic consent renewal settings for recurring data use
